Where You Are vs. Where the Guide Says You Need to Be

Current State

  • 8 skills across 3 folders (FORGE, skills/, Swift Capital/)
  • Narrative skill files — no input/output schemas
  • No negative triggers on any skill
  • Multiple skills do 3–4 jobs each (not atomic)
  • No version numbers, no changelogs
  • No router — agent picks skill by guess
  • No audit log — cannot reconstruct past decisions
  • No golden dataset regression testing
  • Policy thresholds hardcoded in skill prose

Target State (post-overhaul)

  • One skill library: ~/Desktop/FORGE/skills/
  • Every skill: 5-layer compliant, atomic, versioned
  • Router skill runs first, logs dispatch rationale
  • agent.json enforces halt-on-missing-input
  • Append-only audit log per deal
  • 7 golden deals for regression testing
  • Policy Engine (YAML) — one edit propagates to all skills
  • 6 ops questions answerable in 60 seconds
